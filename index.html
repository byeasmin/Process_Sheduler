Scheduling Algorithms

01

FCFS

Non-Preemptive

First Come First Served is the simplest CPU scheduling algorithm that executes processes in the order they arrive.

  • Non-preemptive scheduling
  • Simple implementation
  • First-in-first-out approach
  • No starvation

02

SJF

Non-Preemptive Preemptive

Shortest Job First selects the process with the smallest execution time, minimizing average waiting time.

  • Optimal for minimizing wait time
  • Can be preemptive or non-preemptive
  • Requires burst time knowledge
  • May cause starvation
Learn More Try Demo
03

SRTF

Preemptive

Shortest Remaining Time First is the preemptive version of SJF that dynamically selects processes based on remaining time.

  • Preemptive scheduling
  • Dynamic priority adjustment
  • Better response time
  • Context switching overhead
Learn More Try Demo
04

Priority Scheduling

Non-Preemptive Preemptive

CPU is allocated to the process with the highest priority, ensuring critical tasks are handled first.

  • Priority-based execution
  • Can cause starvation
  • Aging technique available
  • Real-time systems use
Learn More Try Demo
05

Round Robin

Preemptive

Each process gets a fixed time quantum in cyclic order, ensuring fair CPU allocation among all processes.

  • Preemptive scheduling
  • Time quantum based
  • No starvation
  • Good for time-sharing systems

Scheduling Types

Understanding preemptive vs non-preemptive scheduling

Preemptive Scheduling

Preemptive

Processes can be interrupted during execution and moved back to ready queue.

  • Allows higher priority processes to run first
  • Better for time-sharing systems
  • More context switching overhead
  • Examples: Round Robin, SRTF, Priority (Preemptive)

Non-Preemptive Scheduling

Non-Preemptive

Process runs to completion or until it voluntarily yields the CPU.

  • Simpler to implement
  • Less context switching overhead
  • May cause longer waiting times
  • Examples: FCFS, SJF (Non-Preemptive), Priority (Non-Preemptive)
